Formal languages and compilers subject project:

PHP subset language (called IFJ13) interpreter.

Made in collaboration with: Tomáš Bank, Roland Botka, Zdenko Brandejs, Daniil Khudiakov.

Language description

PHP subset. Imperative language with dynamic typing. Case-sensitive.

Data specification

Content Looks like Description
Identifier func _func func_one Func1 funcFunc2 Starts with _A..Za..z., content _A..Za..z0..9.
Variables identifier $var $_var $var_one $Var1 $varVar2 Identifier with $ symbol.
Integer literal 395 0 17 Positive number. 0..9 numbers sequence.
Double literal 13.51 10.5e-3 10e3 4.5E+3 3.4E2 Double value with non-obligatory case-insensitive mantissa.
String literal "word" "more words" "\"Hi!\n\$var\x21’" description below
Logical literal true false Logical value.

String is ASCII 34+ symbols sequence (except " and $ symbols) inside quote marks. String escape sequnces: "\n", "\t", "\\", "\"", "\$" just like in C langauge. "\$" represents dollar symbol. "\xdd" where dd is case-insensitive hexadecimal number in range 00..FF represents symbols with dd ASCII code.

Reserved keywords: else, function, if, return, while. Reserved constants: false, null, true. Reserved keywords and constants can't be used as function or variable identifiers.

Data types: integer, double, string, boolean, null. Interpreter uses dynamic typing, semantic is below.

Structure specification

Main body

<?php
<commands_sequence_with_functions_defintions>

Function definition

function id ( <parameters_sequence> ) {
    <commands_sequence>
}
  • function defintion only inside main body

Commands

$id = <expression>;

if ( <expression> ) {
    <commands_sequence1>
} else {
    <commands_sequence2>
}
while ( <expression> ) {
    <commands_sequence1>
}
$id = <function_name> ( <parameters_sequence> ) ;
return <expression> ;
  • else in if statement is obligatory
  • return statement isn't obligatory
  • function without return statement returns null
  • return in main body doesn't changes interpreter return value (more info about return value below)

=== operator. Result is false if Type 1 isn't equals Type 2. If equals, compares values.

!== operator is === operator negation.

Return codes

Return code Description
1 Laxecial analysis error.
2 Syntax analysis error.
3 Semantic error: undefined function, function redefenition.
4 Semantic error: call function, missing parameter.
5 Semantic error: undeclared variable.
10 Semantic error: divide by zero.
11 Semantic error: type conversion error.
12 Semantic error: expression types comptability.
13 Another semantic error.
99 Internal interpreter error.

You can read error messages at stderr.
